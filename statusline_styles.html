Claude Code 狀態列 — 純文字樣式（無 emoji）

每一條就是顯示在終端機最底下的那一行。挑喜歡的,照卡片下面那句 /statusline … 打就好。

① 直線分隔最乾淨
Opus | demo_app | 18%
打:/statusline 用直線分隔顯示模型、資料夾、context 百分比,不要 emoji
② key:value標籤清楚
model:Opus dir:demo_app branch:main ctx:18%
打:/statusline 用 key:value 格式顯示 model、dir、branch、ctx,不要 emoji
③ 中點分隔(低調)整行淡色
Opus · demo_app · main · 18%
打:/statusline 用中點分隔顯示模型、資料夾、branch、context%,整行用淡灰色,不要 emoji
④ 括號標籤分段感
[Opus] [main] [demo_app] [18%]
打:/statusline 每個欄位都用方括號包起來,顯示模型、branch、資料夾、context%,不要 emoji
⑤ 箭頭路徑風像麵包屑
Opus › main › demo_app › 18%
打:/statusline 用 › 箭頭串接模型、branch、資料夾、context%,不要 emoji
⑥ ASCII 進度條不用方塊字元
Opus [####------] 40% Opus [=======> ] 74% Opus [#########-] 93%
打:/statusline context 用量用 ASCII 進度條 [####----] 顯示,超過 90% 變紅,不要 emoji
⑦ 大寫標籤 + 顏色資訊密度高
MODEL Opus CTX 18% COST $0.42 EFFORT high
打:/statusline 用大寫標籤顯示 MODEL、CTX、COST、EFFORT,標籤用淡色,不要 emoji
⑧ Powerline 色塊反白色塊,最顯眼
Opusmaindemo_app18%
打:/statusline 用 powerline 色塊樣式,每段不同底色顯示模型、branch、資料夾、context%(社群工具 ccstatusline 也能做)
⑨ 多行 · 純文字資訊量最大
Opus demo_app on main +2 ~5 ctx 18% | cost $0.42 | 12m30s
打:/statusline 兩行,第一行模型加資料夾加 git 狀態,第二行 context、花費、時間,不要 emoji
